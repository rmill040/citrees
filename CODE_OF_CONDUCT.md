# Code of Conduct

This project uses a simple standard for participation: be respectful, stay
technical, and help keep the issue tracker useful for research software users
and contributors.

## Expected Behavior

- Be direct, factual, and constructive.
- Assume public issues and pull requests are part of the project record.
- Keep discussion focused on reproducible behavior, documentation, tests, and
  software design.
- Respect maintainers' decisions about project scope.

## Unacceptable Behavior

- Harassment, intimidation, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information, credentials, private datasets, or account
  details without permission.
- Repeated off-topic comments after maintainers have redirected the discussion.

## Reporting

For public project issues, use the GitHub issue tracker:

<https://github.com/rmill040/citrees/issues>

For sensitive conduct or security concerns, email the maintainer listed in
`pyproject.toml`. Reports will be handled by the project maintainers, who may
remove comments, close issues, block participants, or take other actions needed
to preserve a respectful project environment.
