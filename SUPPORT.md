# Support

Use the GitHub issue tracker for support:

<https://github.com/rmill040/citrees/issues>

## Questions and Usage Help

Open a question issue when you need help with installation, estimator usage,
parameter choices, documentation gaps, or reproducing an example. Include the
commands you ran and the package versions involved.

## Bug Reports

Open a bug issue when the package raises an unexpected error, returns incorrect
results, or behaves differently from the documentation. Include a minimal
reproducible example and the full traceback.

## Security or Sensitive Reports

Do not post credentials, private data, AWS account details, or proprietary
datasets in public issues. For sensitive reports, email the maintainer listed in
`pyproject.toml`.

## Support Expectations

This is research software maintained alongside manuscript and benchmark work.
The maintainers will prioritize:

- installation failures
- incorrect results
- documentation that blocks basic use
- reproducibility issues in the public examples and tests
- focused feature requests with clear research use cases

Large benchmark reruns, private data debugging, and custom infrastructure
support may be out of scope for volunteer maintenance.
