\section{Dataset inventories}
\label{app:dataset-inventory}

This appendix lists the datasets used in the benchmark and synthetic feature
recovery analyses. Tables~\ref{tab:benchmark-dataset-inventory-clf}
and~\ref{tab:benchmark-dataset-inventory-reg} list the classification and
regression benchmark datasets. Tables~\ref{tab:synthetic-dataset-inventory-clf}
and~\ref{tab:synthetic-dataset-inventory-reg} list the synthetic classification
and regression designs.

\subsection{Benchmark datasets}

These tables report the number of samples $n$ and features $p$ for each
dataset used in the benchmark.
The packaged parquet snapshots under \texttt{paper/data/} define the exact
datasets analyzed. Source labels identify the public benchmark collections from
which the snapshots were drawn: UCI Machine Learning Repository
\citep{KellyLongjohnNottinghamUCI}, the NIPS 2003 feature selection challenge
\citep{Guyon2004NIPSFeatureSelectionChallenge}, ASU/scikit-feature benchmark
data \citep{Li2018FeatureSelectionDataPerspective}, OpenML
\citep{Vanschoren2014OpenML}, and CoEPrA molecular-property benchmarks
\citep{DemirKavuk2010CoeprA} where noted.

\subsection{Synthetic datasets}

These tables report the number of samples $n$ and features $p$ for each
synthetic design used in the recovery analysis. The Design column gives a short
design label, and the Setting column records the remaining simulation
parameters.

The synthetic datasets were chosen as small controlled examples where the
informative features are known. Each design targets one issue that can matter
for feature selection: correlated predictors, correlated noise, nonlinear signal,
weak signal, high-cardinality noise, redundant features, and an easier and a
harder baseline problem. The same idea is used for the regression datasets,
which also include Friedman~1 and a heteroscedastic design. The parameter
values were set by hand to keep the examples compact and to include one harder
high-dimensional, small-sample case in each task.
