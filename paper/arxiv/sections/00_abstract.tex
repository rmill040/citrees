Greedy decision trees select split variables by optimizing impurity over many
candidate thresholds, creating a multiple-comparisons effect: under a global null,
high-cardinality features are more likely to appear informative by chance,
producing selection bias and unstable rankings. Conditional inference trees
address this by separating variable selection from split-point optimization via
permutation-based hypothesis testing.

We study conditional inference trees and forests that screen features with Monte
Carlo permutation tests before optimizing split points. For this screening step
at a fixed node, we prove finite-sample guarantees: the $+1$ corrected
permutation p-value is super-uniform under an exchangeability null, and
Bonferroni correction yields bounds on both the probability of any false split
and of splitting on any particular feature---without dependence on the number of
candidate thresholds. We also show that, by contrast, CART-style greedy
selection favors features in proportion to their candidate split count under the
same null.

To scale permutation-based trees in practice, we describe implementation
optimizations including adaptive early stopping of permutation tests and
feature scanning. We evaluate feature rankings via a two-stage
rank-then-evaluate benchmark on synthetic ground-truth and real tabular
datasets. A reference implementation is publicly available.
