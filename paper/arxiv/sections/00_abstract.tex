Conditional inference trees (CIT) and conditional inference forests (CIF)
are designed to reduce split-selection bias by testing candidate features before
choosing split thresholds. This matters for feature selection because tree- and
forest-based rankings can otherwise favor features with more possible split
thresholds.
Repeated permutation tests and threshold searches make these methods
computationally expensive. We evaluate CIT and CIF as feature selection methods
using real-data downstream prediction benchmarks, runtime ablation studies, and
synthetic datasets with known informative features.
The formal guarantee is limited to one fixed node. It applies when the candidate
feature set is fixed independently of the node responses and each candidate
feature is tested with the same fixed permutation budget before threshold search.
Under response-label exchangeability within that node,
Bonferroni-corrected $+1$ Monte Carlo $p$-values control the probability of
selecting any tested feature under the nodewise null. In CIF runtime ablations, the
largest measured runtime effects come from adaptive stopping and threshold
search: disabling adaptive stopping raises median runtime 4.0--8.4$\times$
across dataset groups, and exact threshold search is 1.9--10.8$\times$ slower
than histogram-256. These settings change downstream prediction metrics by at
most 0.011 and the fraction of known informative features among top-ranked
features by at most 0.017.
On real datasets, using one selected configuration for each method, CIF ranks
4th among 17
classification methods on 22 datasets and 3rd among 18 regression methods on 8
datasets. Against the reference ctree and cforest baselines, CIF has higher
average downstream classification scores on 22/22 and 19/22 datasets,
respectively. In regression, CIF is higher on 7/8 datasets against
ctree, 7/8 against cforest, and 6/8 against CIT. CIF has its lowest classification mean
ranks at larger selected-feature counts and its lowest regression mean rank at
$k=50$.
Sparse high-$p$ simulations indicate that candidate feature sampling can leave
informative features out of many node-level split tests.
Together, these results support CIF as a practical top-$k$ feature selection
method in these benchmarks and identify candidate feature sampling as an
important control in sparse high-dimensional forests.
