\section{Introduction}
\label{sec:introduction}

Conditional inference introduced an important separation between feature
selection and threshold optimization to reduce split-selection bias. Classical
CART-style trees choose both a feature and a threshold from the same
label-dependent search, which helps explain the well-documented preference for
features with many admissible cutpoints
\citep{Breiman1984ClassificationAndRegressionTrees,LohShih1997SplitSelectionMethods,Strobl2007BiasRFVariableImportance}.
In the conditional-inference framework \citep{Hothorn2006UnbiasedRecursivePartitioning},
Stage~A tests candidate features for association with the response at the
current node and selects among them; Stage~B then optimizes a threshold within
the selected feature. That separation is the right statistical idea, but the
classical implementation can use more computation than necessary when large permutation
budgets and exhaustive threshold scans are applied too uniformly.

This paper takes that narrow fixed-node Stage~A feature-selection object as its
principled core. At a pre-specified node, Stage~A feature selection under the
nodewise complete permutation null defines a local feature-selection object
with a well-defined permutation reference distribution. It is not post-selection
inference for the chosen feature or threshold, and it is not end-to-end
calibration for the full adaptive ranking workflow.

The practical question is then which computations are essential, which add
extra computation, and which preserve useful ranking behavior in practice. CIF
keeps the Stage~A/Stage~B separation while reallocating computation
within it. Adaptive stopping is the
main runtime lever; histogram thresholding separately limits the threshold set;
feature scanning plays a smaller role when early
stopping is active. The remaining ablations are mixed. This is
therefore an optimization-first paper: the benchmark matters here as validation
of CIF, not as the paper's main question.

We evaluate CIF as a feature ranker. We
ask whether, once paired with these practical approximations, CIF remains
useful on real classification rank-then-evaluate benchmarks, whether a similar
pattern appears on a smaller regression panel, and where the
method weakens. On sparse high-dimensional problems, one failure
mode can appear
before threshold search begins: an informative feature may never enter the
candidate set. High-$p$, synthetic, and fixed-design diagnostics
therefore explain these limits, while the real-data benchmark
remains the primary validation layer.

\paragraph{Contributions.}
This paper makes four contributions. First, it proves the fixed-$B$ nodewise
guarantee (\cref{thm:plusone-superuniform,cor:stageA-global-null}): at a
pre-specified node under the complete permutation null, the Stage~A $+1$ Monte
Carlo p-value is super-uniform and Bonferroni controls the probability of any
Stage~A rejection in the tested family. Second, it identifies which practical
controls matter most for speed and score: adaptive stopping provides the largest
speedup, histogram thresholding provides a separate second speedup, and feature
scanning provides a smaller additional speedup when early stopping is active.
The remaining ablations have smaller or less consistent effects. Third, it uses real-data
benchmarks to validate CIF, with classification as the
primary benchmark and regression evaluated on fewer datasets. On real
classification rank-then-evaluate
benchmarks, CIF is the strongest conditional-inference method. It clearly
improves on the historical conditional-inference baselines and CIT, and is
competitive with strong generic ensembles.
Fourth, it uses high-$p$, synthetic, and fixed-design diagnostics to map these
limits: CIF often benefits beyond tiny budgets without usually
needing the full feature set, is better at building useful top-$k$ sets than
at identifying the exact top-ranked feature, and weakens when forest
subsampling leaves informative features out too often.
