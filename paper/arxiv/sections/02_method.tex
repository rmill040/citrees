\section{Algorithmic framework}
\label{sec:method}

At each internal node, conditional inference trees make two decisions. Stage~A
tests candidate features for association with the response and chooses a feature
to split. Stage~B then searches thresholds only for that selected feature and
either returns a valid split or makes the node terminal. In this paper, CIT
denotes the resulting single-tree feature selection method. CIF denotes a
bootstrap ensemble of such trees with candidate feature sampling at each node
and feature rankings formed by aggregating split importances across trees. The
CIF-all variant is used only to study candidate feature coverage: it
disables candidate feature sampling so every nonconstant available feature can
enter Stage~A at each node. The comparison changes candidate feature coverage, the
Stage~A multiplicity correction, and per-node computational cost.

\subsection{Stage~A and Stage~B}
\label{sec:stageA-stageB}

At a node $t$, write $(X_t,Y_t)$ for the samples reaching that node. Let
$F_{t,\mathrm{avail}}$ be the feature set carried into the node by the
tree-growing algorithm, including any feature muting inherited from ancestors.
Let $F_{t,\mathrm{nonconst}} \subseteq F_{t,\mathrm{avail}}$ be the features
that are nonconstant on the node samples. After any \texttt{max\_features}
subsampling, Stage~A considers a set
$F_t \subseteq F_{t,\mathrm{nonconst}}$; its size $m_t=|F_t|$ is the
Bonferroni denominator for that node.

When adaptive stopping is enabled during tree growing, Stage~A can evaluate
features sequentially and terminate before every feature in $F_t$ has been
tested. Let $E_t \subseteq F_t$ denote the evaluated subset. In the exhaustive
fixed-$B$ reference rule, $E_t=F_t$. The fixed-node theorem in
\cref{sec:theory} studies only that exhaustive reference case: $F_t$ and $B$
are label-independent, and every $p_{t,j}$ for $j\in F_t$ is computed with the
fixed budget $B$. Adaptive tree ancestry, feature muting inherited from earlier
splits, and adaptive stopping are not covered by that theorem.

\paragraph{Stage~A (feature selection).}
In the exhaustive fixed-$B$ setting, Stage~A follows the conditional inference
tests of \citet{Hothorn2006UnbiasedRecursivePartitioning} and uses the
permutation statistics of \citet{StrasserWeber1999PermutationStatistics}. It computes a selector
statistic $T^{\mathrm{sel}}_j(X_{t,j},Y_t)$ and permutation p-value $p_{t,j}$
for every feature $j \in F_t$, then selects
%
\begin{equation}
  p_t^{\star,\mathrm{ref}} := \min_{j \in F_t} p_{t,j},
  \qquad
  M_t^\star := \{j \in F_t : p_{t,j}=p_t^{\star,\mathrm{ref}}\}.
\end{equation}
%
Let $\tau_t=\alpha_{\mathrm{sel}}/m_t$ when Bonferroni is enabled and
$\tau_t=\alpha_{\mathrm{sel}}$ otherwise. If
$p_t^{\star,\mathrm{ref}} \ge \tau_t$, Stage~A returns no feature and the node
becomes terminal. If $p_t^{\star,\mathrm{ref}} < \tau_t$, the selected feature
$j_t^{\star,\mathrm{ref}}$ is drawn uniformly from $M_t^\star$ using reservoir
sampling, and the node enters Stage~B.

During adaptive tree growing, Stage~A uses the same decision structure but can
order features by a preliminary score and evaluate only the subset $E_t$. The
resulting score $s_t^\star$ is a search score for tree construction, not an
exhaustive fixed-$B$ p-value covered by \cref{thm:plusone-superuniform}. Under
the exhaustive reference rule with Bonferroni correction,
$\min(1,m_t p_t^{\star,\mathrm{ref}})$ is a conservative complete-null
p-value for the fixed-node Stage~A test. When feature muting is enabled,
features with non-rejecting Stage~A p-values can be removed from descendant
candidate feature pools. Otherwise, apart from deterministic constant-feature pruning,
$F'_{\mathrm{avail}}=F_{t,\mathrm{avail}}$.

\paragraph{Stage~B (threshold search).}
Given a selected feature $j_t^\star$, Stage~B starts from midpoints between
ordered unique values of $X_{t,j_t^\star}$. The exact method uses all such
midpoints. The random method samples a bounded subset of midpoints, while the
percentile and histogram methods choose bounded representative thresholds
derived from the midpoint distribution, with a full-midpoint fallback on very
small nodes. Candidates that would violate
\texttt{min\_samples\_leaf} are filtered out. If no candidate remains, Stage~B
returns no split.

For the remaining finite threshold set $C_{t,j_t^\star}$, Stage~B selects the
best threshold over the candidates. When permutation testing is enabled, the
tested statistic at threshold $c$ is the unweighted sum of child impurities
$\mathrm{Imp}(Y_t^L(c)) + \mathrm{Imp}(Y_t^R(c))$, where $\mathrm{Imp}$ is the
chosen node impurity measure. This statistic is used in a left-tail test, and
the chosen threshold is the candidate with the smallest Stage~B p-value, with
uniform tie breaking. Let
$\tau_t^{\mathrm{split}}=\alpha_{\mathrm{split}}/|C_{t,j_t^\star}|$ when
Bonferroni is enabled for Stage~B and
$\tau_t^{\mathrm{split}}=\alpha_{\mathrm{split}}$ otherwise. If the best
Stage~B p-value is at least $\tau_t^{\mathrm{split}}$, the node becomes
terminal. When Stage~B permutation testing is disabled, the chosen threshold
minimizes the usual weighted child impurity. The final
\texttt{min\_impurity\_decrease} check also uses weighted impurity decrease.
Stage~B is used here to choose the algorithmic split, not to provide a
post-selection p-value with inferential guarantees.

\begin{algorithm}[t]
  \caption{Conditional inference tree training skeleton (Stage~A $\rightarrow$ Stage~B)}
  \label{alg:citree}
  \begin{algorithmic}[1]
    \Require Training data $(X,Y)$, hyperparameters $\theta$
    \Ensure Trained tree $\mathcal{T}$
    \State $p \gets$ number of columns of $X$
    \State \Return \Call{GrowNode}{$X,Y,1,\{1,\dots,p\},\theta$}
    \Function{GrowNode}{$X_t,Y_t,d,F_{\mathrm{avail}},\theta$}
      \If{\Call{Stop}{$X_t,Y_t,d,\theta$}}
        \State \Return \Call{Leaf}{$Y_t$}
      \EndIf
      \State $(j_t^\star, s_t^\star, m_t, F'_{\mathrm{avail}}) \gets \Call{StageA}{X_t,Y_t,F_{\mathrm{avail}},\theta}$
      \If{$j_t^\star = \bot$}
        \State \Return \Call{Leaf}{$Y_t$}
      \EndIf
      \State $(c_t^\star, s_t^{\mathrm{split}}) \gets \Call{StageB}{X_t,Y_t,j_t^\star,\theta}$
      \If{$c_t^\star = \bot$}
        \State \Return \Call{Leaf}{$Y_t$}
      \EndIf
      \State $(X_t^L,Y_t^L,X_t^R,Y_t^R) \gets$ \Call{Partition}{$X_t,Y_t,j_t^\star,c_t^\star$}
      \If{\Call{ImpurityDecrease}{$Y_t,Y_t^L,Y_t^R,\theta$} $< \texttt{min\_impurity\_decrease}$}
        \State \Return \Call{Leaf}{$Y_t$}
      \EndIf
      \State $u \gets$ internal node with split $(j_t^\star,c_t^\star)$ and stored $(s_t^\star,s_t^{\mathrm{split}})$
      \State $u.\mathrm{left} \gets \Call{GrowNode}{X_t^L,Y_t^L,d+1,F'_{\mathrm{avail}},\theta}$
      \State $u.\mathrm{right} \gets \Call{GrowNode}{X_t^R,Y_t^R,d+1,F'_{\mathrm{avail}},\theta}$
      \State \Return $u$
    \EndFunction
  \end{algorithmic}
\end{algorithm}

\subsection{Monte Carlo permutation p-values}
\label{sec:plusone}

Let $T_{\mathrm{obs}}$ be the observed statistic and $T_1,\dots,T_B$ the same statistic
computed on $B$ random label permutations. For a right tail test we use the
Phipson--Smyth $+1$ p-value
\citep{NorthCurtisSham2002EmpiricalPValuesMonteCarlo,PhipsonSmyth2010PermutationPValues}
%
\begin{equation}
  p_{\mathrm{MC}}^{\ge} := \frac{1 + \sum_{b=1}^B \ind\{T_b \ge T_{\mathrm{obs}}\}}{B+1}.
\end{equation}
%
For a left tail test we analogously define
%
\begin{equation}
  p_{\mathrm{MC}}^{\le} := \frac{1 + \sum_{b=1}^B \ind\{T_b \le T_{\mathrm{obs}}\}}{B+1}.
\end{equation}
%
Stage~A uses the right tail convention and Stage~B the left tail convention;
ties count as exceedances in both cases.

\subsection{Forests and runtime hyperparameters}
\label{sec:bootstrap}

CIF combines bootstrap trees and averages their predictions. For feature
selection, it sums each fitted tree's normalized split importances and
renormalizes the forest vector. CIF applies the same Stage~A and Stage~B rules
node by node, while the formal guarantee isolates a single fixed-node Stage~A
test
\citep{Breiman2001RandomForests,HothornBuehlmannDudoitMolinaroVanDerLaan2006SurvivalEnsembles}.
In the CIT/CIF benchmark grid, Bonferroni correction, adaptive stopping,
feature muting, feature scanning, threshold scanning, histogram-256
thresholding, and CIF bootstrap sampling are fixed. Selector and honesty vary
in the CIT/CIF grid.

Runtime is governed by realized counts at each node: features tested in
Stage~A, thresholds generated and retained in Stage~B, permutations evaluated
before stopping, and features carried into the node. Adaptive stopping
changes the number of permutations evaluated for each test
\citep{BesagClifford1991SequentialMonteCarloPValues,Gandy2009SequentialImplementationMonteCarloTests}.
Histogram thresholding changes the Stage~B threshold-set size by replacing
exhaustive cutpoints with binned thresholds. Feature and threshold scanning
reorder candidates under early stopping. Their effect on realized timing is
empirical and is reported in \cref{sec:results-practical}.
Feature muting can reduce the feature pool inherited by descendants.
\Cref{app:methods} records algorithm and reproducibility details,
\cref{app:complexity-details} gives runtime and memory accounting for the
benchmarked tree-growing algorithm, and
\cref{sec:results-practical} reports the measured runtime effects.

\subsection{Feature rankings}
\label{sec:rank-construction}

The experiments evaluate each method through the ranking it produces on the
training fold. CIT and CIF rankings are fitted split-importance rankings, not
rankings of Stage~A p-values. For a fitted CIT, feature $j$ receives the sum,
over internal nodes that split on $j$, of the node impurity decrease: parent
impurity minus the child-size-weighted impurity within that node. The vector is
normalized to sum to one when the sum is positive. CIF sums these tree importance
vectors over trees and then normalizes the forest vector in the same way. Stage~A
determines which features enter threshold optimization, while impurity decrease
scores only the accepted splits by their local reduction in impurity. Larger
importances are ranked earlier, and features never used by a split receive zero
importance.

The other embedded tree and ensemble methods use their fitted importance
vectors in decreasing order: DT, RT, RF, ExtraTrees, XGBoost, LightGBM, and
CatBoost. Filter methods rank features by decreasing dependence score.
Permutation test filters rank by increasing p-value and use the dependence score
as the secondary key. The ctree reference ranks features by split-use counts,
and cforest ranks by unconditional \texttt{partykit::varimp}. Boruta and
recursive feature elimination rank by their fitted rank variables, with smaller
ranks first. Permutation importance and conditional permutation importance rank
by decreasing importance; in this benchmark the latter follows the CPI
calculation described in \cref{sec:experiments-methods}.
When scores or rank variables tie, the
within-tie order is deterministic but method-specific and is used only for
reproducibility.
