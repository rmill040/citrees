\section{Method and software overview}
\label{sec:method}

This section describes the core algorithmic structure implemented by \citrees
and defines the quantities used in the theoretical statements.
Implementation-aligned details and pseudocode are collected in
\cref{app:methods}.

\subsection{Node expansion: Stage A and Stage B}
\label{sec:stageA-stageB}

Consider a fixed node $t$ with data $(X_t, Y_t)$ and a candidate feature set
$F_t$ of size $m_t = |F_t|$.

\paragraph{Stage A (feature screening).}
For each feature $j \in F_t$, \citrees computes a selector statistic
$T^{\mathrm{sel}}_j(X_{t,j}, Y_t)$ (larger indicates stronger association under a
right-tail convention) and a Monte Carlo permutation p-value $p_{t,j}$.
It selects
%
\begin{equation}
  j_t^\star := \argmin_{j \in F_t} p_{t,j}, \qquad p_t^\star := \min_{j \in F_t} p_{t,j},
\end{equation}
%
and proceeds to Stage~B only if $p_t^\star$ is below the configured significance
level (optionally Bonferroni-adjusted by $m_t$).

\paragraph{Stage B (threshold screening).}
Given $j_t^\star$, \citrees constructs a finite threshold set $C_{t,j_t^\star}$
and evaluates split quality over candidates. When permutation testing is
enabled for Stage~B, this is treated as a left-tail test over candidate
thresholds. In this paper, we treat Stage~B p-values as post-selection/adaptive
statistics unless additional selective-inference machinery or sample splitting
is used.

\subsection{Monte Carlo permutation p-values (+1 correction)}
\label{sec:plusone}

Let $T_0$ be the observed statistic and $T_1,\dots,T_B$ the statistics computed
on $B$ i.i.d.\ random label permutations. \citrees uses the Phipson--Smyth (+1)
Monte Carlo p-value
%
\begin{equation}
  p := \frac{1 + \sum_{b=1}^B \ind\{T_b \ge T_0\}}{B+1},
\end{equation}
%
with the inequality direction chosen to match the tail convention.

\subsection{Multi-selector mode (max-statistic)}
\label{sec:multiselector}

In multi-selector mode, multiple association scores are combined within a
feature by taking a maximum across selector statistics inside each permutation.
This yields a single permutation p-value per feature and supports robust
screening when different selectors have complementary power.

\subsection{Forests and bootstrap resampling (classic vs Bayesian)}
\label{sec:bootstrap}

\citrees forests fit $M$ trees and aggregate their predictions. To induce
diversity across trees, the library supports optional bootstrap resampling of
rows. We distinguish:

\paragraph{Classic bootstrap.}
Each bootstrap sample draws $n$ indices with replacement from $\{1,\dots,n\}$
with uniform probabilities $p_i = 1/n$ \citep{Efron1979Bootstrap}.

\paragraph{Bayesian bootstrap.}
The Bayesian bootstrap \citep{Rubin1981BayesianBootstrap} draws a random
probability vector $p = (p_1,\dots,p_n)$ over the $n$ observations, then draws
bootstrap indices with replacement according to $p$. A convenient construction
is
%
\begin{equation}
  w_i \stackrel{\mathrm{i.i.d.}}{\sim} \mathrm{Exp}(1), \qquad p_i = \frac{w_i}{\sum_{j=1}^n w_j},
\end{equation}
%
which yields $p \sim \mathrm{Dirichlet}(1,\dots,1)$. In \citrees, the ``Bayesian
bootstrap'' option uses this normalized-exponential construction to generate
sampling probabilities, and then samples indices with replacement.

For classification forests, \citrees additionally supports stratified and
balanced variants of bootstrapping to control the class proportions within each
bootstrap sample.
