\section{Algorithmic framework}
\label{sec:method}

This section fixes the algorithmic objects used in the rest of the paper.
\Cref{alg:citree} gives the end-to-end tree procedure; appendix material
collects stage-level pseudocode and implementation-aligned details. We
distinguish two execution regimes. The \emph{reference regime}, used only in
\cref{sec:theory}, fixes the permutation budget in advance and exhaustively
evaluates Stage~A at a pre-specified node. The \emph{default engineering
regime} may additionally use early stopping, candidate ordering, feature
muting, threshold subsampling, and forest resampling. Those choices improve
throughput but change the inferential object.

\subsection{Tree growth: Stage~A then Stage~B}
\label{sec:stageA-stageB}

Consider a node $t$ with data $(X_t, Y_t)$ and tested feature family
$F_t \subseteq \{1,\dots,p\}$ of size $m_t$. In the recursive algorithm we
distinguish the \emph{available} feature set passed to the node, the
\emph{descendant-safe} subset of non-constant features at the node, and the
\emph{tested} subset after any \texttt{max\_features} subsampling. Stage~A may
also return an updated feature set to descendants if a pruning heuristic is
enabled.

\paragraph{Stage~A (feature screening).}
For each feature $j \in F_t$, the procedure computes a selector statistic
$T^{\mathrm{sel}}_j(X_{t,j}, Y_t)$ and a Monte Carlo permutation p-value
$p_{t,j}$. It then selects
%
\begin{equation}
  j_t^\star := \operatorname{uargmin}_{j \in F_t} p_{t,j}, \qquad
  p_t^\star := \min_{j \in F_t} p_{t,j},
\end{equation}
%
where $\operatorname{uargmin}$ breaks ties uniformly among minimizers. Stage~B
is entered only if the screening threshold is passed, either at
$\alpha_{\mathrm{sel}}$ or at $\alpha_{\mathrm{sel}}/m_t$ when Bonferroni is
enabled. In the reference regime, the scan over $F_t$ is exhaustive. In the
engineering regime, candidate ordering and early termination may stop the scan
once a significant feature is found, so the selected feature need not be the
global minimum over all tested candidates.

The minimum $p_t^\star$ is not a selection-adjusted p-value for the selected
feature. Under the nodewise complete null, a conservative decision summary for
the Stage~A event is
%
\begin{equation}
  \tilde p_t^{\mathrm{sel}} := \min(1, m_t p_t^\star),
\end{equation}
%
and the Bonferroni rule is equivalently
%
\begin{equation}
  p_t^\star < \alpha_{\mathrm{sel}}/m_t
  \qquad\Longleftrightarrow\qquad
  \tilde p_t^{\mathrm{sel}} < \alpha_{\mathrm{sel}}.
\end{equation}
%
We use $\tilde p_t^{\mathrm{sel}}$ only as a summary of the nodewise
reject/accept decision, not as a calibrated selected-feature p-value.

\paragraph{Stage~B (threshold search).}
Given $j_t^\star$, the procedure constructs a finite threshold set
$C_{t,j_t^\star}$ and evaluates split quality over those candidates. When
permutation testing is enabled for Stage~B, the search is written as a
left-tail test over candidate thresholds. Because the feature and threshold are
selected using the same labels, the reported Stage~B minimum is treated in
this paper as a split-selection score rather than as a calibrated p-value.

\begin{algorithm}[t]
  \caption{Conditional inference tree training (Stage A $\rightarrow$ Stage B)}
  \label{alg:citree}
  \begin{algorithmic}[1]
    \Require Training data $(X,Y)$, hyperparameters $\theta$
    \Ensure Trained tree $T$
    \State $p \gets$ number of columns of $X$
    \State \Return \Call{GrowNode}{$X,Y,1,\{1,\dots,p\},\theta$}
    \Procedure{GrowNode}{$X_t,Y_t,d,F_{\mathrm{avail}},\theta$}
      \If{\Call{Stop}{$X_t,Y_t,d,\theta$}}
        \State \Return \Call{Leaf}{$Y_t$}
      \EndIf
      \State $(j_t^\star, p_t^\star, m_t, F_{\mathrm{avail}}') \gets \Call{StageA}{X_t,Y_t,F_{\mathrm{avail}},\theta}$
      \If{$j_t^\star = \bot$}
        \State \Return \Call{Leaf}{$Y_t$}
      \EndIf
      \State $(c_t^\star, p_t^{\mathrm{split}}) \gets \Call{StageB}{X_t,Y_t,j_t^\star,\theta}$
      \If{$c_t^\star = \bot$}
        \State \Return \Call{Leaf}{$Y_t$}
      \EndIf
      \State $(X_t^L,Y_t^L,X_t^R,Y_t^R) \gets$ \Call{Partition}{$X_t,Y_t,j_t^\star,c_t^\star$}
      \If{\Call{ImpurityDecrease}{$Y_t,Y_t^L,Y_t^R,\theta$} $< \texttt{min\_impurity\_decrease}$}
        \State \Return \Call{Leaf}{$Y_t$}
      \EndIf
      \State $u \gets$ internal node with split $(j_t^\star,c_t^\star)$ and stored $(p_t^\star,p_t^{\mathrm{split}})$
      \State $u.\mathrm{left} \gets \Call{GrowNode}{X_t^L,Y_t^L,d+1,F_{\mathrm{avail}}',\theta}$
      \State $u.\mathrm{right} \gets \Call{GrowNode}{X_t^R,Y_t^R,d+1,F_{\mathrm{avail}}',\theta}$
      \State \Return $u$
    \EndProcedure
  \end{algorithmic}
\end{algorithm}

\subsection{Monte Carlo permutation p-values}
\label{sec:plusone}

Let $T_0$ be the observed statistic and $T_1,\dots,T_B$ the statistics
computed on $B$ random label permutations. For a right-tail test we use the
Phipson--Smyth $+1$ p-value
\citep{NorthCurtisSham2002EmpiricalPValuesMonteCarlo,PhipsonSmyth2010PermutationPValues}
%
\begin{equation}
  p_{\mathrm{MC}}^{\ge} := \frac{1 + \sum_{b=1}^B \ind\{T_b \ge T_0\}}{B+1}.
\end{equation}
%
For a left-tail test we analogously define
%
\begin{equation}
  p_{\mathrm{MC}}^{\le} := \frac{1 + \sum_{b=1}^B \ind\{T_b \le T_0\}}{B+1}.
\end{equation}
%
Stage~A uses the right-tail convention, while Stage~B uses the left-tail
convention. The procedure counts ties as exceedances in both cases.

classical Monte Carlo permutation p-value for fixed-node Stage~A screening, and
as an algorithmic score elsewhere in the tree-growing process.

\subsection{Multi-selector mode}
\label{sec:multiselector}

When multiple Stage~A selector statistics are assigned to the same feature,
the procedure aggregates them through a max-statistic inside each permutation.
This yields a single permutation p-value per feature and supports selector
combinations with complementary power while keeping the screening decision
feature-level.

\subsection{Forests and engineering choices}
\label{sec:bootstrap}

The forest procedure fits multiple trees and aggregates their predictions or
feature-importance summaries. To induce diversity across trees, it supports row
resampling, feature subsampling, and related heuristics. For classification, it
can also use class-conditional bootstrap schemes to control class composition
within a tree's resample.

These options are part of the practical algorithm, not part of the fixed-node
calibration theory. In particular, the root-level guarantee in
\cref{sec:theory} should be read as applying only when the root resampling
scheme preserves the relevant exchangeability assumptions.

\subsection{Practical cost drivers}
\label{sec:complexity}

The runtime experiments are easiest to interpret through four counts at a node:
the number of features carried into the node, the number actually tested in
Stage~A, the number of thresholds generated in Stage~B, and the number of
permutations actually evaluated before any early termination. These counts make
the major tuning knobs easy to read:
\texttt{max\_features} caps Stage~A search width, threshold subsampling caps
Stage~B search width, early stopping and candidate ordering reduce realized
work, and Bonferroni increases the minimum resampling budget needed for
rejection.

The main-text point is qualitative rather than asymptotic: most engineering
knobs reduce realized work without changing the underlying inferential target,
while Bonferroni changes both the decision rule and the computational burden.
\Cref{app:complexity-details} gives the implementation-aligned complexity and
memory formulas.
