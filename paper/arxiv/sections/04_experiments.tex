\section{Experimental design}
\label{sec:experiments}

\subsection{Benchmark setup}
\label{sec:experiments-protocol}

The benchmark protocol uses 5 random seeds and fivefold cross validation within
each seed. Classification uses logistic regression, support vector machine
(SVM), and k-nearest neighbors (KNN) as downstream models and reports balanced
accuracy, the mean recall across classes. Regression uses ridge regression,
support vector regression (SVR), and KNN and reports $R^2$. The main tables use
the standard values $k \in \{5,10,25,50,100\}$.
Downstream learner settings are fixed: logistic regression uses balanced class
weights and 1,000 maximum iterations; SVM uses an RBF kernel with $C=1$,
\texttt{gamma=scale}, and balanced class weights; KNN uses 5 distance-weighted
neighbors; ridge uses $\alpha=1$; and SVR uses an RBF kernel with $C=1.0$,
$\epsilon=0.1$, and \texttt{gamma=scale}.

For each seed and fold, feature standardization is fit on the training fold and
applied to the held-out fold before ranking. The ranking method is then fit on
the standardized training fold only and produces an ordered list of features as
described in \cref{sec:feature-rankings}. The held-out fold is evaluated using
the top-$k$ features from that training fold ranking. For each $k$, a second
standardization is fit on the selected training fold features, the same transform
is applied to the held-out fold, and a downstream model is fit on that selected
feature set.

The downstream-evaluation pipeline also records the full feature set $k=p$ for
every dataset. For datasets with more than 100 features, the separate high-$p$
analysis adds fixed values $k \in \{150,200,300,500,750,1{,}000\}$ and
fractional values
$\lceil 0.25p\rceil,\lceil 0.5p\rceil,\lceil 0.75p\rceil$, retaining only
values that are at most $p$. The main tables average only the standard values
of $k$.

\subsection{Benchmark comparisons}
\label{sec:experiments-analysis-plan}

The benchmark uses three averaging steps. First, within each
$(\text{dataset}, \text{downstream model}, k)$ combination, scores are averaged
over folds and seeds. Second, each dataset score averages those fold-and-seed
means over downstream models and the standard $k$ values shared by the methods
being compared. Third, the tables average dataset scores over datasets.

The real-data classification benchmark contains 23 datasets. Comparisons
involving the partykit ctree and cforest references use 22 datasets because
those references were not run on \texttt{dexter};
comparisons among methods with \texttt{dexter} results use all 23. The
14-dataset classification analysis requires all 17 methods, all three downstream
learners, and all five standard values of $k$; it is used for bootstrap
intervals and omnibus rank testing. Regression has 8 datasets; the analogous
rank test uses the 6 datasets with all methods, learners, and standard $k$
values present.

The trajectory analyses do not average over the number of selected features; they
report results separately at each standard value of $k$. At
$k = 5, 10, 25, 50, 100$, these trajectory analyses use
$22, 21, 15, 15, 14$ classification datasets and $8, 8, 7, 7, 6$ regression
datasets. Pairwise CIF comparisons stratified by $k$ use the dataset counts
reported with the corresponding rows. The 14- and 6-dataset analyses retain
only datasets with every method, every downstream learner, and every standard
value of $k$.

\subsection{Datasets}
\label{sec:experiments-datasets}

The benchmark on real datasets includes 23 classification datasets and 8
regression datasets. The separate high-$p$ analysis restricts attention to the 15
classification datasets and 6 regression datasets with more than 100 features.
The full inventories for real and synthetic datasets are listed in
\Cref{app:dataset-inventory}, which also summarizes the synthetic designs and
their parameter choices.

The synthetic recovery analysis uses 8 classification and 8 regression datasets
with known informative features and controlled noise structure. A separate
candidate feature sampling study uses $n=250$ and varies
$p \in \{100, 500, 1{,}000\}$ and
$n_{\mathrm{informative}} \in \{1,2,5,10\}$. That study compares CIT,
a CART decision tree (DT), a random threshold tree (RT), CIF, CIF with all
features available at each node (CIF-all), random forests (RF), and ExtraTrees
(ET), using 1,000 trees for forest methods.

\subsection{Compared methods}
\label{sec:experiments-methods}

The benchmark includes three types of feature selectors. Permutation test filters
use univariate dependence scores to rank features: multiple correlation (MC) and
randomized dependence coefficient (RDC) for classification, and Pearson
correlation (PC), distance correlation (DC), and RDC for regression. The RDC
filter is an RDC-style randomized-feature dependence score adapted from
\citet{LopezPaz2013RDC}: it keeps the empirical-CDF random feature map but uses
the maximum absolute pairwise correlation across projected columns instead of a
canonical-correlation solve. Distance correlation follows
\citet{Szekely2007DistanceCorrelation}. Embedded methods use tree
or ensemble importance scores: CART decision trees (DT), randomized trees (RT),
CIT, CIF, random forests (RF)
\citep{Breiman2001RandomForests},
ExtraTrees \citep{Geurts2006ExtraTrees}, XGBoost
\citep{ChenGuestrin2016XGBoost}, LightGBM \citep{Ke2017LightGBM},
CatBoost \citep{Prokhorenkova2018CatBoost}, and the reference
conditional-inference implementations ctree and cforest from
\texttt{partykit}
\citep{RPartykitPackage,Hothorn2006UnbiasedRecursivePartitioning,HothornZeileis2015Partykit}.
Wrapper baselines use auxiliary random-forest fits to score feature subsets: Boruta
\citep{Kursa2010Boruta}, permutation importance (PI)
\citep{Breiman2001RandomForests,Strobl2007BiasRFVariableImportance}, a
stratified conditional-permutation approximation (CPI approx.) motivated by
conditional permutation importance
\citep{DebeerStrobl2020ConditionalPermutationImportanceRevisited}, and
random-forest recursive feature elimination (RF-RFE)
\citep{Guyon2002RFE,Pedregosa2011ScikitLearn}. The CPI approximation trains a
random forest on a training split, forms validation-set strata from features
correlated with the target feature, and permutes the target feature within
those strata. It is a benchmark baseline, not a claim to reproduce every detail
of published CPI estimators. In the implementation used here, CPI approx. uses a
20\% validation split, conditions on features with absolute correlation at least
0.5 with the target feature, forms five-bin strata, and scores a 100-tree random
forest after within-stratum permutation.

\subsection{Configuration selection}
\label{sec:experiments-repro}

Some methods, including CIT, CIF, XGBoost, LightGBM, ctree, and cforest, were
run under multiple settings. Permutation test filters used one setting per
statistic. For each task, the paper reports one selected configuration per
method family, chosen by mean score over real datasets, downstream models, and
the standard values $k \in \{5,10,25,50,100\}$. Methods with only one run
configuration are treated as fixed. The selected setting is then held fixed
across datasets for that task. Because selection uses the same real-data
benchmark reported below, the scores, intervals, and rank tests evaluate
settings chosen from this benchmark rather than configurations fixed before the
benchmark.

As a sensitivity check, we repeat the configuration choice in
leave-one-dataset-out (LODO) form on the 14-dataset classification check. Each
held-out dataset is evaluated under settings chosen from the other 13 datasets.
The downstream models, $k$ values, and averaging within each dataset stay the
same.

For the synthetic analysis, configuration choice is separate from the benchmark
on real datasets. Each method is assigned the setting with the highest average
recovery of the true informative features over $k = 5, 10, 25, 50,$ and $100$.

For CIT and CIF, the default benchmark settings use
adaptive sequential stopping, Bonferroni correction, feature muting, feature
scanning, threshold scanning, and histogram thresholding with 256 bins. The
benchmarked CIT and CIF variants differ in selector and honesty; CIF uses the
forest form with bootstrap sampling held fixed. The CIF hyperparameter ablation
changes one setting at a time.
