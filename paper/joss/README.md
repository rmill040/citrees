# JOSS submission

This directory contains the JOSS paper source for `citrees`.

## Files

- `paper.md`: JOSS Markdown manuscript.
- `paper.bib`: bibliography for `paper.md`.
- `paper.pdf`: local Inara draft PDF; ignored until the submission has real
  JOSS metadata.
- `jats/`: local JATS files generated by Inara; ignored.

## Build

JOSS builds papers with the Open Journals tooling. With Docker installed, run
from the repository root:

```bash
docker run --rm \
  --volume "$PWD/paper/joss:/data" \
  --user "$(id -u):$(id -g)" \
  --env JOURNAL=joss \
  openjournals/inara
```

Do not commit the draft PDF or JATS output until the JOSS metadata is real.

The GitHub Actions workflow `.github/workflows/joss-draft-pdf.yml` builds a
draft from `paper/joss/paper.md` on paper changes and uploads it as
`joss-paper`.

## Format requirements

The paper should follow the current JOSS paper-format guidance:

- Begin with a non-specialist summary.
- Stay between 750 and 1750 words.
- Include `Summary`, `Statement of need`, `State of the field`,
  `Software design`, `Research impact statement`, `AI usage disclosure`,
  `Acknowledgements`, and `References`.
- Keep API documentation in the package docs rather than the paper.
- Use real repository, release, archive, and JOSS identifiers only after they
  exist; do not add placeholder URLs or DOIs to `paper.md`.
