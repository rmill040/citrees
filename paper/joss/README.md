# JOSS submission

This directory contains the draft **Journal of Open Source Software** (JOSS)
paper derived from the `citrees` package and the full arXiv manuscript in
`paper/arxiv/`.

## Files

- `paper.md`: JOSS Markdown manuscript.
- `paper.bib`: bibliography for `paper.md`.
- `paper.md.txt`: saved copy of the JOSS author-format instructions.
- `paper.pdf`: draft PDF generated by Inara.
- `jats/`: JATS files generated by Inara.

## Saved reference

- `paper.md.txt` is the JOSS author instructions saved **verbatim** from:
  - `https://joss.readthedocs.io/en/latest/_sources/paper.md.txt`

## Build

JOSS builds papers with the Open Journals tooling. With Docker installed, run
from the repository root:

```bash
docker run --rm \
  --volume "$PWD/paper/joss:/data" \
  --user "$(id -u):$(id -g)" \
  --env JOURNAL=joss \
  openjournals/inara
```

The local draft PDF includes placeholder DOI, review, repository, archive,
editor, reviewer, and submission metadata until JOSS creates the review issue
and publication metadata.

The GitHub Actions workflow `.github/workflows/joss-draft-pdf.yml` follows the
Open Journals draft-PDF action documented by JOSS. It builds this paper from
`paper/joss/paper.md` and uploads `paper/joss/paper.pdf` as the `joss-paper`
artifact on paper changes.

## Format requirements

The paper should follow the current JOSS paper-format guidance:

- Begin with a non-specialist summary.
- Stay between 750 and 1750 words.
- Include `Summary`, `Statement of need`, `State of the field`,
  `Software design`, `Research impact statement`, `AI usage disclosure`,
  `Acknowledgements`, and `References`.
- Keep API documentation in the package docs rather than the paper.
- Use real repository, arXiv, release, and archive identifiers only after they
  exist; do not add placeholder URLs or DOIs to `paper.md`.
